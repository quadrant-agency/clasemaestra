Para el desarrollador

Guía para construir el sistema de clasemaestra, paso a paso.

Guía paso a paso, a prueba de todo: comandos exactos para copiar, prompts detallados para Claude Code y una checklist de "cómo verificar" en cada día. La meta: que la plataforma quede idéntica al prototipo. Pensada para que la ejecute alguien sin perfil técnico, en jornadas de ~8 horas.

01 · Objetivo

Objetivo y alcance

Construir el sistema completo y funcional de clasemaestra tal como está definido en el prototipo navegable, en el menor tiempo posible y con calidad de producción.

✓ Sí entra en este build

  • Las 24 pantallas del prototipo (público, maestra, admin), funcionales y conectadas.
  • Autenticación, roles y base de datos.
  • Pagos y suscripciones con Stripe (alta, cambio de plan/modalidad, anual, add-ons, pausa de verano, cancelación, dunning, garantía 7 días).
  • Editor en plataforma (tipo documento), "Mi estilo" (kit de marca) y aplicar a todas.
  • Comunidad, programa de aportes y referidas.
  • Backend de admin (contenidos, planes, flywheel, señal, CRM, correos).
  • Sistema de diseño clasemaestra desde el día 1.

✕ NO entra (por ahora)

  • Los agentes de IA que producen contenido (Agente 0–4) y la generación automática de planeaciones.
  • La producción del contenido pedagógico en sí (eso se carga manualmente / vendrá después).
Importante: el sistema debe quedar listo para recibir ese contenido (estructura de bundles, actividades por grado/campo/semana, modalidades), aunque el contenido se cargue manualmente desde el admin mientras tanto.

La fuente de verdad es el prototipo

Cuando haya cualquier duda de comportamiento, copy, jerarquía o flujo, gana el prototipo. Cada día de esta guía enlaza a las pantallas exactas que debes replicar: prototipo.html.

03 · Método

Cómo usar esta guía con Claude Code

Cada día trae uno o más prompts listos para pegar en Claude Code. El flujo recomendado para no perder detalle:

  • 1. Crea un CLAUDE.md en la raíz del repo (ver sección "Convenciones"). Pégale el stack, los design tokens, las reglas de negocio y la liga al prototipo. Claude Code lo lee en cada sesión.
  • 2. Trabaja un día = una rama. git checkout -b dia-04-publico. Pega el prompt del día. Revisa, prueba, commitea.
  • 3. Abre el prototipo al lado. Antes de aceptar la pantalla, compárala contra su deep-link. La fuente de verdad es el prototipo.
  • 4. Verifica los criterios de aceptación de cada día antes de pasar al siguiente. No avances con deuda.
  • 5. Pídele a Claude Code que escriba pruebas de los flujos críticos (auth, checkout, webhooks, gating por plan).
Regla de oro de los prompts: siempre termina pidiéndole a Claude Code que (a) respete los design tokens, (b) compare contra el deep-link del prototipo, y (c) liste qué criterios de aceptación quedaron cumplidos y cuáles no.

Tipografía y forma

  • Tipografía: Helvetica Now Display Bold (si hay licencia); si no, Archivo (Google Fonts) como sustituto web.
  • Wordmark: "clase" en gris + "maestra" en carbón, minúsculas, bold.
  • Titulares: mayúscula de oración (no todo en minúsculas, no TODO MAYÚSCULAS).
  • Radios: ~12–16px. Sombras suaves. Estilo limpio y cálido.
  • Mascota Kipi: 4 poses en /assets/. Decorativa; nunca presentada como "un agente que trabaja por ti".
Regla crítica: el rojo es solo para alertas/errores/acciones destructivas. El CTA principal siempre es cian con texto carbón.

Regla de marca de producto

Cero menciones de IA en contenido visible al usuario. El posicionamiento es "creado y revisado por expertos en educación". La IA puede existir internamente, pero nunca en copy público ni en la app de la maestra.

★ Contrato compartido

Formato de contenido por bloques

Este es el formato del campo content de una actividad y de una personalización. Lo usan 5 pantallas (actividad, editor I y II, personalizaciones y "Mi estilo"). Defínelo una sola vez como un tipo de TypeScript en src/lib/blocks.ts y que todo lo use; así nada se desincroniza.

content (jsonb){ "version": 1, "blocks": [ { "id": "b1", "type": "heading", "level": 2, "text": "Inicio" }, { "id": "b2", "type": "paragraph", "text": "Texto editable de la planeación..." }, { "id": "b3", "type": "list", "style": "bullet", "items": ["Material 1", "Material 2"] }, { "id": "b4", "type": "image", "url": "https://.../foto.jpg", "alt": "Material" } ] }
typeCampos
headinglevel (1-3), text
paragraphtext
liststyle (bullet/number), items []
imageurl, alt
Una personalización es una copia de este mismo content con los cambios de la maestra. El estilo (color/tipografía/encabezado) NO va aquí: vive en brand_kits y se aplica al renderizar.

  • Los avisos de Stripe mantienen el acceso sincronizado con el pago, y la garantía de 7 días reembolsa.
